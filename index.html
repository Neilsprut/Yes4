About us

Acerta identifies defects in safety-critical software

Demonstrate your system is safe

Acerta was created to make standard compliance and certification a simple, cost efficient, and evidence driven process.

How does it work?

Acerta uses logs of system activity, or traces, to extract key insights about system behavior. These insights are then used as evidence that the system complies with safety requirements. Acerta makes it easy to demonstrate that the system functions correctly during run time, and can produce a ready-to-use report for certification.

No need for customization

Operating systems, hardware silicon, and runtime environments provide different means to collect traces. Acerta does not impose restrictions on the traces collected by the system. It uses any type of trace format and adapts to the type of information that is available. As such, Acerta does not require any extra work or customization from the manufacturer, therefore reducing the time and cost necessary to get the quality evidence.


Our Work
Current pilot projects

QNX Car Case Study

The car platform is a comprehensive case study geared towards automotive software and systems research. The case study offers several Advanced Driver Assistance Systems (ADAS) implemented on an RC car model running on a treadmill.

The setup offers a rich set of safety-critical applications with timing constraints in a controlled environment. Available ADAS functions include lane departure warning, lane keeping assistance, and adaptive cruise control.

Acerta was used to analyze the behavior of the driver assistance systems at run time. Through the collected system traces, we identified important system scheduling properties, interrupt behavior, communication properties, and performance metrics.
